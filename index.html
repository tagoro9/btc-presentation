Decentralized digital currency established in 2009 which allows people to anonymously exchange amounts of bitcoins without the need of a central authority in charge of transactions.

Decentralized

The Bitcoin consists on a P2P network. No authority controls Bitcoin. Coins are created using an algorithm and given to those who collaborate to verify transactions. Everyone has the history of all transactions from begginning of time.

Anonymous

Every Bitcoin user has a public / private key pair. Public keys can be seen as an account number to which other users send money. There is no way to map these public keys to identities

Can't rollback

Transactions are irreversible, they can only be refunded by the user receiving the funds.

No mandatory transaction fees

Unlike credit card payments, merchants do not need to pay for every transaction performed although people can use transactions fees to get their transaction confirmed sooner.

21 million bitcoins

Maximum number of bitcoins that will ever exist. It is limited by design and just like gold it becomes a limited resource.

Double spending

How to avoid double spending? That is, how to avoid that anyone uses their money twice. To answer that I need to introduce blocks and the block chain

Blocks

Sets of transactions that are appended to a block chain. To be added to this chain, a block needs to be valid.

Block validation

    • All transactions must be valid
    • Block header hash lower than network difficulty

Block mining

  • Header nonce parameter
  • Winning the lottery
  • Reward as proof of work
  • Reward halfed every 210,000 blocks

Difficulty

  • Target block hashes must be lower than
  • Network parameter adjusted periodically
  • 1 Block every 10 minutes rate

Block chain

  • Shared database containing all transactions
  • Every peer has a copy
  • Transactions are valid when in block chain
  • The deeper, the more difficult to modify
  • Avoid double spending

Is it secure?
